# 医院就诊挂号系统

前后端分离的医院就诊挂号系统，使用 WebSocket 实现实时通信。

## 功能特点

### 患者端
- 在线挂号（填写个人信息、选择科室和挂号类型）
- 实时查看排队状态和排队号码
- 预计等待时间显示
- 状态变化实时通知（等待中、就诊中、已完成）
- 断线自动重连

### 医院端
- 实时查看所有挂号患者
- 按科室和状态筛选
- 叫号功能
- 完成就诊
- 取消挂号
- 统计数据展示（总人数、等待中、就诊中、已完成）
- 新患者挂号实时提醒

## 技术栈

- 后端：Node.js + Express + WebSocket
- 前端：原生 HTML + CSS + JavaScript
- 实时通信：WebSocket
- 数据存储：内存存储（可扩展为数据库）

## 安装步骤

1. 安装依赖：
```bash
npm install
```

2. 启动服务器：
```bash
npm start
```

或使用开发模式（自动重启）：
```bash
npm run dev
```

3. 访问系统：
- 医院端：http://localhost:3000/hospital.html
- 患者端：http://localhost:3000/patient.html

## 使用说明

### 患者挂号流程
1. 打开患者端页面
2. 填写个人信息（姓名、性别、年龄、电话等）
3. 选择就诊科室和挂号类型
4. 提交挂号
5. 查看排队号码和预计等待时间
6. 等待叫号

### 医院管理流程
1. 打开医院端页面
2. 查看实时挂号队列
3. 点击"叫号就诊"呼叫患者
4. 患者就诊完成后点击"完成就诊"
5. 可按科室和状态筛选患者

## 特色功能

- **实时同步**：患者端和医院端通过 WebSocket 实时同步数据
- **智能排队**：自动计算预计等待时间
- **状态管理**：完整的患者状态流转（等待中→就诊中→已完成）
- **断线重连**：网络断开后自动重连
- **数据持久化**：患者端使用 localStorage 保存挂号信息
- **通知提醒**：支持浏览器通知（需授权）

## 注意事项

- 当前版本使用内存存储，服务器重启后数据会丢失
- 建议在生产环境中使用数据库（如 MongoDB、MySQL）
- 可根据实际需求调整每个挂号类型的就诊时间
- 支持多个医院端和患者端同时连接
